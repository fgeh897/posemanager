# Sims4 动作管理器项目架构分析

## 📋 项目概览

这是一个**Sims4 游戏 MOD**，功能是帮助玩家管理游戏中的"动作(Pose)"。主要功能包括：
- 💾 收藏和管理喜欢的动作
- 👥 按作者分类查看动作  
- 📦 管理动作包
- 🔄 多种浏览模式切换

---

## 🏗️ 项目整体架构

```
项目结构：
┌─────────────────────────────────────┐
│   UI 第一层（主菜单）                 │
│   - 收藏动作包 Favorite Packs       │
│   - 作者列表 Creators               │

└────────────┬──────────────────────┘
             │
             ↓
┌─────────────────────────────────────┐
│   UI 第二层（子菜单）                 │
│   - 显示具体动作列表                 │

└────────────┬──────────────────────┘
             │
             ↓
┌─────────────────────────────────────┐
│   点击处理逻辑                        │
│   - 收藏/取消收藏                    │
│   - 切换浏览模式                     │
│   - 导航返回                         │
└─────────────────────────────────────┘
```

## 🔑 核心数据结构

### 1. **收藏列表（全局变量）**
```python
FAVORITE_POSES      # 收藏的动作名称列表
FAVORITE_CREATORS   # 收藏的作者名称列表
FAVORITE_PACKS      # 收藏的动作包ID列表
```

### 2. **缓存字典（性能优化）**
```python
GLOBAL_ALL_POSES_CACHE     # 存储所有动作及其信息
GLOBAL_ALL_CREATORS_CACHE  # 存储所有作者及其动作包
GLOBAL_ALL_PACKS_CACHE     # 存储所有动作包
```

### 3. **状态标志（界面控制）**
```python
CURRENT_MODE            # 当前浏览模式 (0=普通, 1=收藏, 2=追踪)
FAV_PACKS_VIEW_MODE     # 包管理模式 (0=浏览, 1=管理, 2=删除)
CREATOR_VIEW_MODE       # 作者管理模式 (0=收藏, 1=全部, 2=包列表, 3=删除)
```

## 📁 文件系统

项目会创建 3 个 JSON 文件来保存用户的收藏：

```
mod_directory/
├── my_favorite_poses.json      # 收藏的动作
├── my_favorite_creators.json   # 收藏的作者
└── my_favorite_packs.json      # 收藏的包
```

**数据格式示例：**
```json
{
  "my_favorite_poses.json": ["pose_001", "pose_002"],
  "my_favorite_creators.json": ["Creator_Name_1", "Creator_Name_2"],
  "my_favorite_packs.json": ["pack_id_1", "pack_id_2"]
}
```

## 🔄 工作流程

### 流程 1：玩家打开动作选择菜单

```
玩家点击"选择动作"
    ↓
游戏调用 PoseByPackInteraction.picker_rows_gen()
    ↓
我们的 custom_pack_rows_gen_safe() 接管
    ↓
生成 4 个特殊选项（收藏包、作者、全部动作 + 普通包）
    ↓
玩家看到菜单
```

### 流程 2：玩家选择"我的收藏包"

```
玩家点击"我的收藏包"
    ↓
pack_type = 'FAV_PACKS'
    ↓
显示子菜单：
  - "管理收藏" → 进入管理模式
  - "删除收藏" → 进入删除模式  
  - "全部收藏的包" → 列表显示
    ↓
玩家选择一个包
    ↓
selected_pose_pack = 选中的包
    ↓
进入该包的动作列表
```

### 流程 3：数据加载与缓存

```
程序启动
    ↓
load_favorites() 从 JSON 读取收藏数据
    ↓
build_cache_safe() 扫描所有游戏中的动作包
    ↓
将动作包分类存入三个缓存字典
    ↓
后续操作直接使用缓存（快速）
```

## 🎮 三种浏览模式

| 模式 | 值 | 说明 | 用途 |
|-----|-----|------|------|
| **MODE_NORMAL** | 0 | 普通浏览 | 直接浏览和选择动作 |
| **MODE_FAV** | 1 | 收藏模式 | 点击动作时收藏/取消收藏 |
| **MODE_TRACE** | 2 | 追踪模式 | 从收藏夹中选动作，自动找到原始包 |

**举例说明：**
- 你在"我的收藏"里看到 pose_001，点击它
- MODE_FAV: 系统从收藏列表中移除 pose_001
- MODE_TRACE: 系统自动找到 pose_001 来自哪个包，跳转到那个包

## 💡 设计亮点

### 1. **四层菜单系统**
- L1: 选择菜单分类（包、作者、动作）
- L2: 显示具体项目列表
- L3: 子菜单（管理/删除）
- L4: 具体动作列表

### 2. **双向导航**
- `CAME_FROM_L1_FAVS`: 记录是否来自收藏包
- `CAME_FROM_CREATOR`: 记录是否来自作者列表
- 玩家可以随时返回上一级

### 3. **缓存机制**
- 首次加载时扫描所有动作包（耗时）
- 之后使用缓存（快速）
- `CACHE_BUILT` 标志确保只扫描一次

### 4. **数据持久化**
- 每次修改后自动保存到 JSON
- 游戏重启后数据恢复

### 5. **本地化支持**
- 所有文本使用哈希值而不是硬编码字符串
- 支持多种游戏语言

## 📝 总结

这个 MOD 本质上是一个**菜单系统**，通过以下方式实现：

1. **拦截游戏菜单** → 注入自己的菜单项
2. **构建多层菜单** → 分类管理动作
3. **记录用户选择** → 保存到 JSON 文件
4. **重新显示菜单** → 实现循环交互

整个系统围绕"**状态管理**"和"**事件处理**"构建，是一个经典的 UI 框架设计。
